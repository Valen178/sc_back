=== CAMBIOS NECESARIOS PARA PRODUCCIÓN ===

1. SUPABASE:
   - Si tienes un proyecto separado para producción:
     * Actualizar SUPABASE_URL con la URL de tu proyecto de producción
     * Actualizar SUPABASE_ANON_KEY con la clave anónima de producción
   - Si usas el mismo proyecto:
     * No es necesario cambiar nada, pero asegúrate de tener las políticas de seguridad correctamente configuradas

2. PORT:
   - En producción, muchos servicios (como Heroku) asignan el puerto dinámicamente
   - El código ya está preparado para usar el puerto asignado por el servicio de hosting
   - No necesitas cambiar nada en el .env, ya que usa PORT=3000 como fallback

3. JWT_SECRET:
   - IMPORTANTE: Genera una nueva clave secreta para producción
   - Usa el comando: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   - Nunca uses la misma clave de desarrollo en producción
   - Guarda esta clave de forma segura y no la compartas

4. JWT_EXPIRES_IN:
   - Actualmente configurado a 24h
   - Ajusta según tus necesidades de seguridad en producción
   - Formatos válidos: "20h", "2d", "1w"

5. GOOGLE OAUTH:
   - En Google Cloud Console:
     a. Añade tu dominio de producción en "Authorized JavaScript origins":
        Ejemplo: https://tuapp.com
     
     b. Añade tu URL de callback de producción en "Authorized redirect URIs":
        Ejemplo: https://tuapp.com/auth/google/callback
   
   - En tu .env de producción:
     * Actualiza GOOGLE_CALLBACK_URL con tu URL de producción
     * GOOGLE_CLIENT_ID y GOOGLE_CLIENT_SECRET pueden ser los mismos si usas el mismo proyecto de Google
     * Para mayor seguridad, considera crear un proyecto separado en Google Cloud para producción

=== EJEMPLO DE .ENV PARA PRODUCCIÓN ===

# Supabase configuration
SUPABASE_URL=https://tu-proyecto-prod.supabase.co
SUPABASE_ANON_KEY=tu-clave-produccion

# Server configuration
PORT=3000

# JWT configuration
JWT_SECRET=nueva-clave-secreta-generada-para-produccion
JWT_EXPIRES_IN=24h

# Google OAuth
GOOGLE_CLIENT_ID=id-cliente-google-produccion
GOOGLE_CLIENT_SECRET=secreto-cliente-google-produccion
GOOGLE_CALLBACK_URL=https://tuapp.com/auth/google/callback

=== NOTAS DE SEGURIDAD ===

1. Nunca comitees el archivo .env al repositorio
2. Mantén diferentes archivos .env para desarrollo y producción
3. Usa un gestor de secretos en producción si es posible
4. Haz backups seguros de tus credenciales de producción
5. Rota periódicamente las claves sensibles como JWT_SECRET
