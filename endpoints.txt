=== ENDPOINTS DE LA API ===

BASE URL: http://localhost:3000

=== GESTIÓN DE PUBLICACIONES ===

1. Crear Nueva Publicación
   POST /posts
   Headers: {
     "Authorization": "Bearer <token>"
   }
   Body: {
     "text": string,      // Opcional si se proporciona URL
     "url": string       // Opcional si se proporciona texto
   }

2. Obtener Todas las Publicaciones
   GET /posts
   Query Parameters:
     - page: número de página (default: 1)
     - limit: número de posts por página (default: 10)
   Response: {
     "message": "Posts retrieved successfully",
     "data": {
       "posts": [...],
       "pagination": {
         "current_page": number,
         "total_pages": number,
         "total_posts": number,
         "posts_per_page": number
       }
     }
   }

3. Obtener Publicaciones de Usuario Específico
   GET /posts/user/:userId
   Query Parameters:
     - page: número de página (default: 1)
     - limit: número de posts por página (default: 10)

4. Eliminar Publicación
   DELETE /posts/:postId
   Headers: {
     "Authorization": "Bearer <token>"
   }
   Notas: 
   - Solo el propietario o un admin puede eliminar la publicación
   - Se requiere autenticación
Versión: 1.0.0
Última actualización: 09/09/2025

=== CONVENCIONES ===

1. Autenticación:
   - La mayoría de los endpoints requieren token JWT
   - Formato: Authorization: Bearer <token>
   - Tokens expiran en 24 horas

2. Respuestas:
   - Formato: application/json
   - Códigos HTTP estándar
   - Mensajes descriptivos en errores

3. Validaciones:
   - Campos requeridos indicados
   - Tipos de datos especificados
   - Referencias válidas verificadas

=== AUTENTICACIÓN ===

1. Registro de Usuario
   POST /auth/signup
   Body: {
     "email": string,
     "password": string,
     "role": "user", // por defecto
     "phone_number": number
   }

2. Inicio de Sesión
   POST /auth/login
   Body: {
     "email": string,
     "password": string
   }

3. Inicio de Sesión con Google
   POST /auth/google
   Body: {
     "token": string // Token de Google
   }

4. Completar Perfil
   POST /auth/complete-profile
   Headers: {
     "Authorization": "Bearer <token>"
   }
   Body: {
     // Para perfil de atleta
     "name": string,
     "last_name": string,
     "birthdate": date,
     "height": number,
     "weight": number,
     "location_id": number,
     "sport_id": number
   }
   // O
   Body: {
     // Para perfil de agente
     "name": string,
     "last_name": string,
     "description": string,
     "location_id": number
   }
   // O
   Body: {
     // Para perfil de equipo
     "name": string,
     "job": string,
     "description": string,
     "sport_id": number,
     "location_id": number
   }

     "password": string,

     "phone_number": number
   }

4. Actualizar Usuario
   PUT /admin/users/:id
   Body: {
     // Campos a actualizar
   }

5. Eliminar Usuario
   DELETE /admin/users/:id

6. Cambiar Rol de Usuario
   PATCH /admin/users/:id/role
   Body: {
     "role": "user" | "admin"
   }

       "name": string,

       "description": string,
       "location_id": number
     }
     // O para perfil de equipo
     "profileUpdates": {

   POST /api/swipe - Dar like/dislike

=== NOTAS ===

- Todos los endpoints de administrador requieren autenticación y rol de admin
- Los endpoints de perfil y completar perfil requieren autenticación
- Los endpoints de lookup son públicos y no requieren autenticación
- Las respuestas incluyen mensajes de error descriptivos en caso de fallos
- Todos los endpoints devuelven respuestas en formato JSON
